Rendering¶

Rendering Device¶

Textures can be baked on the CPU or GPU. To use the GPU, Cycles must be configured to use a compatible device:

Edit > Preferences > System > Cycles Render Devices

Resolution¶

The final output texture resolution used for baking.

Anti-Aliasing¶

Improves edge quality (mainly along geometry and UV island borders) by baking at a higher resolution and scaling down to the final output size.

Options:

  • MSAA x2: Bakes at 2× resolution, then downscales.
  • MSAA x4: Bakes at 4× resolution, then downscales.
  • MSAA x8: Bakes at 8× resolution, then downscales.

Note: Higher MSAA values increase bake time and memory usage. Padding also becomes slower because it is computed on the higher-resolution result before downscaling.

Padding¶

Padding (edge padding / bleed) extends pixels beyond UV island borders. This prevents visible seams when mipmaps and texture filtering sample outside UV shells.

Little Bakery supports three padding algorithms:

  • Fast: Expands pixels in 4 directions. Very fast, especially for small padding values.
  • Chamfer: Expands pixels in 8 directions. Slightly slower but generally more accurate than Fast.
  • EDT: Euclidean Distance Transform. Computes true distance to the island boundary and produces the most consistent padding, typically at higher cost.
