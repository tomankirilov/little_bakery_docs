Bake Targets¶

Little Bakery supports multiple texture outputs ("bake targets"). Bake targets are the final output texture types per render set. Each render set can enable multiple bake targets at once.

There are two kinds of bake targets:

  • Built-in: Provided by Little Bakery.
  • Custom: User-defined targets using a custom material.

Below is a list of the available bake targets and their settings.

Normal¶

Bakes a normal map.

Options:

  • Space-Tangent: Tangent-space normal maps are commonly used for real-time rendering (game assets).
  • Space-Object: Object-space normal maps store normals in object space and are mainly used for static meshes or offline workflows.
  • Swizzle: Remaps the normal directions X, Y, Z to output channels R, G, B (useful for engine/channel conventions).

Ambient-Occlusion¶

Bakes an Ambient Occlusion (AO) map: how much ambient light is blocked by nearby geometry. Creases and tight gaps bake darker; open areas bake lighter. Often used as a grounding/masking texture.

Options:

  • Ray Count: Number of AO rays cast per texel. Higher values reduce noise but increase bake time.
  • Render Samples: Cycles sample count used for the bake. Higher values reduce noise but increase bake time.
  • Distance: Maximum ray length used to find occluders. Lower values focus on small creases; higher values capture broader occlusion.
  • Contrast: Adjusts the AO intensity distribution to make dark/light separation stronger or softer.

Curvature¶

Bakes a Curvature map: where the surface is convex (edges/ridges) or concave (valleys). Useful for edge wear, dirt buildup, stylized shading, and as a general-purpose mask.

Options:

  • Exponent (Default 2.2): Applies a gamma-style correction so the curvature is easier to blend in color space.
    With 2.2, neutral curvature (flat / no curvature) maps to 50% gray, which is convenient for blending and masking.
    With 1.0, the neutral point is not at 50% gray.
  • Contrast: Applied to the raw curvature data before baking (pre-encoding). This increases or decreases separation between edges and flats without compressing the underlying data, while preserving a stable neutral midpoint (50% gray after the exponent step).

Thickness¶

Bakes a Thickness map: approximate distance from the surface to the opposite side of the mesh (how “thick” the object is). Commonly used for translucency or subsurface-scattering masks.

Options:

  • Ray Count: Number of rays cast per texel. Higher values reduce noise but increase bake time.
  • Render Samples: Cycles sample count used for the bake. Higher values reduce noise but increase bake time.
  • Distance: Maximum thickness to search for. Lower values emphasize thin features; higher values capture thicker volumes.

Position¶

Generates a texture storing the X, Y, Z position of each object.

Bakery Position¶

Similar to Position, but normalized to the 0–1 range.

Random Island¶

Bakes a random value per Source Mesh. Useful for masking and variation.

Color Attribute¶

Bakes a color attribute per Source Mesh. The default attribute name is "Color".

Options:

  • Color Attribute: Default attribute name to bake. If the attribute does not exist on the Source Mesh, the result will be pure black.
    The attribute can be overridden per Source Mesh.

Custom¶

Custom bake targets allow using a custom material on the Source Mesh.

Options:

  • Material: The material to use for baking.
  • Bake Type: Bake type used by the Cycles render engine.
